## ACT 6 — Incident Injection & Root Cause (2 phút)

**Người trình bày**: Member D

**Nói**: "Đây là phần quan trọng nhất — simulate production incident và debug theo đúng quy trình."

### Scenario: RAG Slow

**Inject incident:**
```bash
# T2
curl -X POST http://127.0.0.1:8000/incidents/rag_slow/enable
```

**Verify:**
```bash
curl http://127.0.0.1:8000/health
# → "incidents": {"rag_slow": true, ...}
```

**Gửi requests trong khi incident active:**
```bash
python scripts/load_test.py --concurrency 2
```

**Quan sát dashboard** — P95 latency tăng lên ~2700ms, badge chuyển vàng

**Debug flow — giải thích 3 bước:**

**Bước 1 — Metrics:** "Dashboard báo P95 latency tăng từ 1.2s lên 2.7s. Đây là signal."

**Bước 2 — Traces:** Mở Langfuse, filter traces gần đây → click trace có latency cao
```
Trace mới:
├── agent.run()              2743ms
│   ├── [RAG retrieve]       2538ms  ← ĐÂY! RAG chiếm 92%
│   └── [LLM generate]        181ms
```

**Nói**: "Trace waterfall ngay lập tức chỉ ra: RAG span = 2538ms — bất thường. LLM vẫn bình thường."

**Bước 3 — Logs:** Tìm log của trace đó:
```bash
# Dùng correlation_id từ trace để grep log:
cat data/logs.jsonl | python -m json.tool | grep "req-xxxxx"
```

**Root Cause:** "Tìm ra `rag_slow` toggle đang ON trong incidents.py — `time.sleep(2.5)` được gọi mỗi request."

**Fix:**
```bash
curl -X POST http://127.0.0.1:8000/incidents/rag_slow/disable
python scripts/load_test.py --concurrency 2
# → Latency trở về bình thường
```

## Q&A PREPARATION

### Câu hỏi thường gặp theo từng thành viên:

**Cho Member A (Correlation ID):**
- *"Tại sao cần clear_contextvars() ở đầu mỗi request?"*
  > Vì FastAPI dùng async workers — nếu không clear, correlation_id của request trước có thể leak sang request sau đang xử lý trên cùng worker. Lỗi rất khó debug.

- *"Tại sao format req-8hex thay vì UUID đầy đủ?"*
  > UUID đầy đủ 36 chars làm log khó đọc. 8 hex chars (4 tỷ combinations) đủ unique cho một session demo/lab.

**Cho Member B (PII):**
- *"Tại sao CCCD pattern phải đặt trước phone_vn?"*
  > CCCD là 12 digits bắt đầu bằng 0. Phone_vn cũng bắt đầu bằng 0 và match 10-11 digits. Nếu phone chạy trước, nó sẽ consume 10 digits đầu của CCCD và để lại 2 digits — CCCD pattern không còn match được nữa.

- *"Pattern address_vn có false positive không?"*
  > Có thể — "đường" trong câu thông thường như "con đường phía trước" sẽ match. Trade-off: false positive ít hại hơn false negative (leak địa chỉ thật). Trong production nên dùng NLP-based approach.

**Cho Member C (SLO/Alerts):**
- *"Tại sao P95 thay vì mean latency?"*
  > Mean bị kéo bởi outlier. Nếu 95 requests xong trong 1s nhưng 5 requests mất 30s, mean = ~2.5s nhưng trải nghiệm của 5% user đó rất tệ. P95 đo "worst case của đa số".

- *"Symptom-based alert nghĩa là gì?"*
  > Alert khi triệu chứng ảnh hưởng user ("latency > 5s") thay vì alert theo nguyên nhân ("CPU > 80%"). CPU cao không nhất thiết ảnh hưởng user; latency cao thì chắc chắn ảnh hưởng.

**Cho Member D (Traces/Incidents):**
- *"Langfuse observe decorator làm gì?"*
  > Wrap hàm `agent.run()`, tự động bắt start/end time, input/output, và gửi lên Langfuse. Không cần viết instrumentation code thủ công.

- *"Làm sao biết trace nào là chậm?"*
  > Filter trong Langfuse: sort by `latency DESC`, xem top 5 chậm nhất. Click vào → waterfall span chỉ ra ngay component nào mất nhiều thời gian.

**Cho Member E (Dashboard):**
- *"Dashboard lấy data từ đâu?"*
  > Từ `/metrics` endpoint của FastAPI. Endpoint này gọi `metrics.snapshot()` — hàm đọc in-memory lists (latencies, costs, tokens, quality scores) và compute aggregates. Auto-refresh 5 giây.

- *"Tại sao không dùng Grafana?"*
  > Grafana cần Prometheus exporter + config phức tạp hơn. Với lab này, HTML dashboard đủ để demonstrate 6 required panels và SLO thresholds. Trong production thì Grafana tốt hơn.
